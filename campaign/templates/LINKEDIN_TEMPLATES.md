## Key Design Decisions

### 1. Tauri v2 Over Electron

**Why Tauri v2?**
- Native webview (no bundled Chromium)
- Sandboxed Rust backend (security)
- Tiny bundle size (8MB vs 120MB+)
- Fast IPC (Tauri commands)

**Result:** 50MB RAM vs 200MB+ for Electron

### 2. Rust Backend Over Node.js

**Why Rust?**
- Memory safety without garbage collection
- Zero-cost abstractions
- Excellent async runtime (tokio)
- Rich SSH ecosystem (russh)

**Result:** Stable, secure, and performant

### 3. SQLite Over File-based Storage

**Why SQLite?**
- Embedded database (no external deps)
- ACID compliance
- Good performance for structured data
- Easy encryption layer

**Result:** Reliable local storage with encryption

## Tauri Commands

Frontend → Backend communication via Tauri commands:

```rust
#[tauri::command]
async fn ssh_connect(
    window: Window,
    session_id: String,
    host: String,
    port: u16,
    auth: AuthMethod,
) -> Result<(), String> {
    // Connect to SSH server
    let session = SshSession::connect(&host, port, auth).await
        .map_err(|e| e.to_string())?;

    // Store session in memory
    SESSIONS.lock().await.insert(session_id.clone(), session);

    // Notify frontend
    window.emit("ssh-connected", session_id).map_err(|e| e.to_string())?;

    Ok(())
}
```

Frontend usage:

```typescript
import { invoke } from '@tauri-apps/api/core';
```

## Security Considerations

### 1. Sandboxed Backend

Tauri's sandbox ensures:
- No arbitrary file system access
- No network access without permission
- No shell command execution
- Encrypted IPC communication

### 2. Credential Storage

- Master password required to unlock vault
- AES-256-GCM encryption at rest
- Keys stored in OS keychain
- Secure memory handling (zero after use)

### 3. SSH Key Management

- Keys stored encrypted in vault
- Support for multiple key formats (RSA, ED25519)
- Passphrase protection with Argon2id
- Secure key generation with `ring`
